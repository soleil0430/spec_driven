# FileBookmark - 사용 가이드

## 개요

FileBookmark는 Unity 에디터에서 **자주 사용하는 에셋(파일/폴더)에 빠르게 접근**할 수 있도록 도와주는 에디터 확장 도구입니다.

프로젝트 창에서 깊이 중첩된 폴더를 탐색하거나, 자주 사용하는 파일을 찾아 헤매는 시간을 줄이고, 작업 효율성을 향상시킵니다.

## 주요 기능

- **프리셋 기반 관리**: 작업 환경별로 다른 에셋 그룹을 관리
- **드래그 앤 드롭**: Project 창에서 직접 드래그하여 즉시 추가
- **계층 구조 지원**: 폴더 추가 시 하위 구조 자동 로드
- **GUID 기반 추적**: 파일 이름/경로 변경에도 추적 유지
- **Missing 항목 관리**: 삭제된 에셋 자동 감지 및 일괄 제거
- **빠른 접근**: 싱글 클릭으로 선택, 더블 클릭으로 열기

---

## 시작하기

### 창 열기

Unity 에디터 메뉴에서:
```
Tools > File Bookmark
```

### 화면 구성

```
┌──────────────────────────────────────────────┐
│  [Default] [+] [-] [Edit] [Clear]           │ ← 프리셋 관리
├──────────────────────────────────────────────┤
│  Drag files/folders here      Missing: 0    │ ← 툴바
├──────────────────────────────────────────────┤
│  📁 Scripts                                  │
│    📄 GameManager.cs                         │
│  📁 Prefabs                                  │ ← 북마크된 파일들
│    📄 Player.prefab                          │
└──────────────────────────────────────────────┘
```

## 기본 사용법

### 1. 파일/폴더 추가하기

1. Unity의 **Project 창**에서 파일 또는 폴더 선택
2. **FileBookmark 창**으로 드래그
3. 마우스 버튼 놓기

**팁**: 폴더를 추가하면 하위 구조가 자동으로 로드됩니다.

### 2. 파일 접근하기

- **싱글 클릭**: Project 창에서 해당 에셋 선택 및 위치 표시
- **더블 클릭**: 에셋 열기 (스크립트는 IDE에서, Prefab은 에디터에서)
- **우클릭**: 항목 제거 메뉴

### 3. 프리셋 관리

#### 프리셋 생성
- 상단 **[+]** 버튼 클릭
- 작업 성격에 맞게 프리셋 구성 (예: "UI 작업", "게임플레이", "사운드")

#### 프리셋 전환
- 상단의 프리셋 버튼 클릭
- 현재 선택된 프리셋은 파란색으로 표시

#### 프리셋 이름 변경
- **[Edit]** 버튼 클릭
- 새 이름 입력

#### 프리셋 삭제
- **[-]** 버튼 클릭 (최소 1개는 유지되어야 함)

#### 프리셋 전체 비우기
- **[Clear]** 버튼 클릭 (되돌릴 수 없으니 주의!)

### 4. 항목 제거

- 항목 우클릭 → **Remove** 선택
- FileBookmark에서만 제거되며, 실제 에셋은 삭제되지 않습니다

## Missing 항목 처리

### Missing이란?

프로젝트에서 삭제되거나 이동된 에셋을 의미합니다. FileBookmark는 GUID로 에셋을 추적하지만, 에셋이 삭제되면 더 이상 접근할 수 없습니다.

### 식별 방법

- ⚠️ 경고 아이콘 표시
- 흐린 붉은색 텍스트
- 툴팁에 "Missing: [경로]" 표시

### 제거 방법

**개별 제거**: 항목 우클릭 → **Remove** 선택

**일괄 제거**: 툴바 우측 **[Remove All]** 버튼 클릭

## 활용 예시

### 예시 1: UI 개발

```
프리셋: "인벤토리 UI"
├─ Assets/Scripts/UI/Inventory/
├─ Assets/Prefabs/UI/Inventory/
└─ Assets/Resources/Sprites/UI/
```

작업 중 FileBookmark에서 스크립트, Prefab, 스프라이트를 빠르게 열어가며 작업합니다.

### 예시 2: 멀티 시스템 개발

```
프리셋 1: "전투 시스템"
프리셋 2: "상점 시스템"
프리셋 3: "퀘스트 시스템"
```

시스템 전환 시 프리셋을 전환하여 빠른 컨텍스트 스위칭이 가능합니다.

### 예시 3: 리팩토링

```
프리셋: "리팩토링 - 아이템 시스템"
├─ ItemManager.cs
├─ ItemData.cs
├─ InventorySystem.cs
└─ ItemDatabase.asset
```

리팩토링 대상 파일들을 한 곳에 모아 작업 범위를 명확히 합니다.

## 팁 & 트릭

### 프리셋 활용

- 프로젝트 초기에 주요 폴더를 프리셋에 추가
- 작업 타입별 프리셋 구성 ("긴급 수정", "매일 사용", "테스트")

### 파일 vs 폴더

- **폴더 추가**: 전체 시스템을 한 번에 관리
- **파일 추가**: 특정 파일만 빠르게 접근

### 정기적인 관리

- 주기적으로 Missing 항목 확인 및 제거
- 프로젝트 정리 후 반드시 확인

### 프리셋 이름 규칙

명확하고 설명적인 이름 사용:
- 좋은 예: "UI - 인벤토리", "Scripts - Core", "Audio - BGM"
- 나쁜 예: "임시", "작업중", "test"

### 순서 정렬

- 자주 사용하는 항목을 상단에 배치
- 루트 레벨 항목은 드래그로 재정렬 가능

## 주의사항

### 설정 파일

- **위치**: `Assets/FileBookmark/Editor/FileBookmarkSettings.txt`
- **형식**: JSON (자동 저장)
- **버전 관리**: `.gitignore`에 추가 권장 (로컬 전용)

### 성능

매우 큰 폴더(수천 개 파일)를 추가하면 로딩 시간이 증가할 수 있습니다.
→ 필요한 하위 폴더만 선택적으로 추가하세요.

### 백업

중요한 프리셋 구성은 `FileBookmarkSettings.txt` 파일을 복사하여 백업하세요.

## 트러블슈팅

### Q: 드래그 앤 드롭이 작동하지 않습니다
**A**: 
- Unity 에셋만 추가 가능합니다 (Project 창의 에셋)
- Hierarchy의 GameObject는 추가할 수 없습니다

### Q: 항목을 클릭해도 반응이 없습니다
**A**:
- Missing 항목일 수 있습니다 (경고 아이콘 확인)
- 에셋이 실제로 존재하는지 Project 창에서 확인하세요

### Q: 프리셋이 저장되지 않습니다
**A**:
- `FileBookmarkSettings.txt` 파일 권한 확인
- Unity 에디터 재시작 시도
- 콘솔에 오류 메시지가 있는지 확인

### Q: 재정렬이 작동하지 않습니다
**A**:
- 루트 레벨 항목만 재정렬 가능합니다
- 폴더 내부의 파일은 파일 시스템 순서를 유지합니다

## 키보드 단축키

현재 FileBookmark는 기본 단축키를 제공하지 않습니다.

Unity 에디터 설정에서 직접 추가할 수 있습니다:
1. `Edit > Shortcuts...`
2. "File Bookmark" 검색
3. 원하는 키 조합 설정

**권장**: `Ctrl/Cmd + Shift + B` (Bookmark의 B)
